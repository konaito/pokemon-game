# ゲームテーマ設計書

## 中心となる「問い」

**「記憶を失った世界で、それでも繋がりを信じられるか？」**

---

## テーマ: 忘却と記憶 — つながりの再発見

### コンセプト

この世界では、かつて人間とモンスターは深い絆で結ばれていた。しかし「大忘却（グレート・フォゲット）」と呼ばれる謎の現象により、人々はモンスターとの記憶を大部分失ってしまった。モンスターたちは野生に戻り、人間は彼らを「危険な野生動物」として恐れるようになった。

主人公は、モンスターとの絆が「当たり前」だった時代を知らない。しかし、ある出来事をきっかけに最初のモンスターと出会い、「かつて存在した絆」を自分の手で再発見していく旅に出る。

### テーマの3層構造

```
表層:  「モンスターと仲良くなって冒険する楽しさ」
中層:  「忘れられたものを取り戻す旅」
深層:  「記憶がなくても、新しい絆は生まれるか」
```

- **子供向けの表層**: 冒険、捕獲、バトルの楽しさ
- **物語としての中層**: なぜモンスターとの絆が失われたのか、どうやって取り戻すのか
- **哲学的な深層**: 過去の関係性を知らなくても、今ここで新しい関係性を築けるか

### テーマの体験方法

テーマは直接語らず、ゲームプレイを通じて体験させる:

1. **序盤**: NPCたちはモンスターを怖がっている。「あんな危ないものに近づくな」と言われる
2. **中盤**: 各地のジムリーダーたちは「忘れられた絆」を独自の方法で守ってきた人々だと分かる
3. **転換点**: 悪の組織が「大忘却」を意図的に起こした/利用しているという真実
4. **終盤**: 伝説のモンスター = 記憶の守護者であり、世界の記憶を取り戻す鍵

### 各キャラクターとテーマの関係

| キャラクター | テーマとの関係 |
|------------|--------------|
| **主人公** | 「記憶なき世代」。だが偏見なくモンスターと向き合える |
| **ライバル** | 「記憶を恐れる者」。過去に囚われ変化を拒む → 最後に受容する |
| **博士** | 「記憶を研究する者」。大忘却の真相を追い続けている |
| **悪の組織** | 「忘却を利用する者」。人々の記憶喪失を利用して支配する |
| **ジムリーダー** | 「記憶を守る者」。各地で密かにモンスターとの絆を受け継いできた |
| **伝説のモンスター** | 「記憶そのもの」。世界の記憶を内包する存在 |

## テーマの「裏返し」= 悪の組織の思想

悪の組織「オブリヴィオン団」は、大忘却を「進化」と呼ぶ。

> 「モンスターに依存していた旧時代は弱かった。忘却こそが人間の独立だ。記憶を取り戻す？ なぜ過去にしがみつく？ 我々は忘却の先に、新しい秩序を築く」

プレイヤーへの問いかけ:
- 忘れてしまったものを、わざわざ取り戻す必要があるのか？
- 新しい関係は、過去の関係の代わりになるのか？
- それとも、全く別の新しいものなのか？

**答えは物語が提示する**: 主人公は過去の記憶を取り戻すのではなく、**今ここで新しい絆を作ることで世界を変える**。過去への執着でも忘却でもなく、「今を生きること」が答え。

## ゲームタイトル案

**MONSTER CHRONICLE** (モンスタークロニクル)

- Chronicle = 年代記、記録
- 「失われた記録を取り戻す旅」「新しい記録を作る旅」の二重の意味
- 既にタイトル画面に "MONSTER CHRONICLE" が実装済み

## テーマから導かれる設計原則

1. **世界設計**: 人間とモンスターの関係が「断絶」した世界。かつての共存の痕跡が遺跡として残る
2. **町の設計**: 各町はモンスターとの関係の「段階」を表す（恐怖→無関心→好奇心→共存→尊敬）
3. **ジムリーダー**: 「記憶の守り手」。各タイプのモンスターとの古い絆を守ってきた
4. **御三家**: 博士の研究で唯一人間に懐いている3匹。大忘却前の絆の「残り火」
5. **伝説のモンスター**: 記憶を司る存在。2体（記憶/忘却、あるいは過去/未来）
6. **エンディング**: 世界の記憶は完全には戻らないが、主人公たちの新しい絆が「次の時代」を開く
